# 🚀 构建应用前必读

## ⚠️ 重要：配置SMTP邮件服务

在构建应用之前，**必须**配置SMTP邮件服务，否则用户将无法注册账户。

### 快速配置（3分钟）

1. **复制配置文件**
   ```bash
   cp smtp-config.example.json smtp-config.json
   ```

2. **获取163邮箱授权码**
   - 访问 https://mail.163.com
   - 登录邮箱：`ruiding_support@163.com`
   - 点击 **设置** → **POP3/SMTP/IMAP**
   - 开启 **SMTP服务**
   - 生成 **授权码**（不是邮箱密码！）

3. **填写授权码**
   
   编辑 `smtp-config.json`：
   ```json
   {
     "host": "smtp.163.com",
     "port": 465,
     "user": "ruiding_support@163.com",
     "pass": "这里填写授权码"
   }
   ```

4. **验证配置**
   ```bash
   # 本地测试
   npm start
   # 尝试注册账户，检查是否能收到邮件
   ```

5. **构建应用**
   ```bash
   # Windows
   npm run build:win
   
   # macOS
   npm run build:mac
   ```

### 常见问题

**Q: 为什么需要配置SMTP？**
A: 应用需要发送邮箱验证码来：
- 确保账户唯一性（一个邮箱一个账户）
- 验证用户身份
- 支持密码重置功能

**Q: 不配置会怎样？**
A: 用户无法注册新账户，会提示"邮件服务未配置"

**Q: 授权码是什么？**
A: 授权码是126邮箱为第三方应用生成的专用密码，不是邮箱登录密码

**Q: 授权码会泄露吗？**
A: 不会，`smtp-config.json` 已加入 `.gitignore`，不会上传到GitHub

### 安全提示

- ✅ `smtp-config.json` 只在本地存在
- ✅ 不会被提交到Git仓库
- ✅ 打包时会包含在应用中（用户看不到）
- ⚠️ 不要在公开场合展示授权码
- ⚠️ 定期更换授权码
